# 稿子菌 v7.0 · 学术投稿管理工具

全功能学术投稿审稿管理工具——跟踪论文投稿进度、管理审稿任务、记录期刊评价。

纯静态 HTML 单文件应用，数据存储在浏览器本地或绑定本地文件夹（仓库），无需服务器。

## 功能概览

### 📝 投稿管理
- 记录论文投稿信息：标题、期刊、状态、身份（第一作者/通讯/合作者）
- 优先级标记、标签分类、搜索筛选
- **多轮投稿关联**：同一篇论文改投多刊，自动关联成作品链
- 按状态/身份/期刊/优先级分组浏览

### 📈 投稿进度（时间线）
- 每篇论文独立时间线，记录投稿全流程节点
- 每个时间线节点支持**上传审稿意见**和**审稿意见回复**文件
- 自动计算各阶段间隔天数
- 投稿总天数统计

### 📪 审稿管理
- 记录你作为审稿人的审稿任务
- 审稿邀请、截止日期、审稿决定（接收/小修/大修/拒稿）
- 审稿时间线跟踪

### 📊 统计仪表盘
- 投稿趋势折线图 + 状态分布饼图
- 按年份/月份/期刊筛选

### 📝 期刊评价
- 记录期刊评价信息：影响因子、投稿体验、审稿周期
- 支持中文（CSSCI/北大核心/经济Top）和英文（ABS/SSCI/中科院）等级体系
- 评价笔记 + 社区评价汇总

### 🔔 投稿提醒
- 超过设定天数（默认14天）未更新的投稿自动提醒
- 可暂停/恢复单篇提醒

### 📎 文件管理
- 按论文组织文件：稿件、附件、其他
- 支持 PDF、Word、Excel、图片等格式
- Office 文件在线预览（需启动本地 server.py）

### 📦 数据管理
- **仓库模式**：绑定本地文件夹，数据自动同步到文件系统
- **JSON 导入/导出**：备份和恢复数据
- **仓库数据导出**：打包为 ZIP 下载

## 快速开始

### 方式一：直接用浏览器打开

1. 下载 `稿子菌.html`
2. 用 Chrome/Edge 浏览器打开（推荐，支持 File System Access API）
3. 数据存储在浏览器 localStorage，可随时导出备份

### 方式二：本地服务器（可选，启用 Office 文件预览）

```bash
# 在同一目录下放置 稿子菌.html 和 server.py
python server.py
# 访问 http://localhost:8080/稿子菌.html
```

### 方式三：Android 安装

安装 `稿子菌-v7.2.apk`，首次启动选择仓库文件夹或新建。

## 稿子菌仓库结构

绑定本地文件夹作为"仓库"后，数据自动按以下结构存储：

```
稿子菌仓库/
├── _稿子菌配置.json          ← 全局配置（标签、设置）
├── 投稿/                     ← 论文数据（JSON）
│   └── {ID}_{论文标题}.json
├── 审稿/                     ← 审稿数据（JSON）
│   └── {ID}_{期刊}.json
├── 文件/                     ← 按论文组织的上传文件
│   └── {论文名称}/
│       ├── 稿件/              ← 论文 PDF/Word
│       ├── 附件/              ← 补充材料
│       ├── 审稿意见/           ← 审稿意见文件
│       └── 审稿意见回复/        ← 审稿意见回复
└── 评价/                     ← 期刊评价（JSON）
    └── {ID}_{期刊名}.json
```

## 技术说明

- 纯前端 HTML/CSS/JS，单文件 ~170KB
- 使用 Chart.js 渲染图表（CDN 加载）
- 数据存储：localStorage + File System Access API（仓库模式）
- 无需后端服务器，无数据上传风险
- 推荐浏览器：Chrome / Edge（Chromium）>= 86
- 字体：霞鹜文楷（CDN 加载，可选）
